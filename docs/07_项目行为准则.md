# 07 项目行为准则（代码规范 / 掘金免费版 / 企业级准则）

状态：Draft v0.1
适用范围：`main.py`、`strategy/`、`config/strategy.yaml`、`tests/`、`docs/`

---

## 1. 总原则

### 必须
- 代码以**解耦、可维护、可回测、可实盘降级**为优先目标。
- 条款必须可验证（代码路径、测试断言、运行日志、报表结果）。
- 真实行为以代码和测试为准，文档必须与其一致。

### 禁止
- 在策略层与 GM 接口层混写业务逻辑。
- 写“不可执行”的口号式规范。

### 建议
- 每次改动附最小证据：改动文件 + 验证命令 + 验证结果。

---

## 2. 代码规范（解耦与可维护性）

### 必须
- 维持分层边界：
  - `engine` 只做流程编排；
  - `pipelines` 只做策略/风控决策；
  - `gm` 是唯一行情与交易 IO 边界；
  - `reporting` 只做报表。
- `pipelines` 不得直接依赖 GM SDK。
- 核心流程保持固定顺序：`flush -> trend -> risk -> entry -> daily`。
- 新增逻辑必须可被 smoke 测试覆盖。

### 禁止
- 跨层调用（如在 `pipelines` 里直接下单）。
- 在回调函数中堆叠难以维护的策略细节。

### 建议
- 遵循 PEP 8 与类型注解。
- 将“可变参数”统一收敛到配置层，避免散落硬编码。

代码证据：
- `strategy/engine.py`
- `strategy/gm/callbacks.py`
- `tests/test_smoke_pipeline.py`

## 3. 掘金免费版实战流程图（文字版）

### 3.1 研究阶段
1. 识别可用数据与受限能力：仅依赖可稳定获得的数据（OHLCV、主连映射）。
2. 定义可验证假设：趋势过滤、突破确认、风险约束都要有可回放指标。

### 3.2 回测阶段
1. 使用最小回测模板：
   - `main.py --mode BACKTEST --config config/strategy.yaml`
2. 免费版回测窗口策略：
   - 回测起止时间默认设置为**官方允许的最大可回测时间窗**；
   - 超限时按“最近可用完整窗口”回退，不改变策略纪律。
3. 必跑稳健性测试：
   - 样本外测试；
   - 滚动窗口测试；
   - 参数扰动测试（风控与入场核心参数）。

### 3.3 试运行阶段
1. 小资金上线门槛：先单品种/小仓位运行，先验证执行稳定性再扩容。
2. 风控阈值：触发日损/回撤状态即暂停新开仓并进入人工复核。
3. 人工接管规则：日志异常、回调异常、报表异常任一触发均进入人工接管。

### 3.4 数据与执行注意事项
- 夜盘与日盘阈值分离。
- 换月必须清空挂起信号与候选平台状态。
- 必须考虑滑点、成交约束与流动性差异造成的回测-实盘偏差。

### 3.5 每周迭代节奏
- 固定检查：数据完整性、信号质量、风险触发分布、报表一致性。
- 版本管理：参数改动与逻辑改动分离记录，保留对比结果。

代码证据：
- `strategy/gm/callbacks.py`
- `strategy/gm/contract_roll.py`
- `strategy/gm/market_data.py`
- `config/strategy.yaml`

## 4. 数据使用合规与稳健性检查

### 必须
- 明确数据来源与字段：`eob/open/high/low/close/volume`。
- 明确频率边界：5m 用于执行，1h 用于趋势过滤。
- 明确时间对齐：信号执行必须发生在信号创建 bar 之后。
- 检查 `shift/rolling/merge` 的未来函数风险。

### 禁止
- 使用任何未来信息直接或间接参与当前 bar 决策。
- 在数据不足时强行计算并执行信号。

### 建议
- 夜盘跨日按会话区间判断，不按自然日简单截断。
- 对缺失字段提供降级路径，但不放松实盘纪律。

代码证据：
- `strategy/gm/market_data.py`
- `strategy/pipelines/entry.py`
- `strategy/pipelines/chan_5m.py`

## 5. GM 回调兼容与稳定性

### 必须
- `run` 参数覆盖核心字段：`token/strategy_id/mode/backtest_start_time/backtest_end_time`，可选 `serv_addr`。
- callback kwargs（`init/on_bar/on_order_status/on_execution_report/on_error`）必须先做签名能力检测再注入。
- 当凭据缺失时必须安全降级为本地模式。
- 当 SDK 不支持 callback kwargs 时，仅回退 kwargs，不丢失核心运行参数。

### 禁止
- 直接假设目标环境一定支持完整 callback kwargs。
- 回调异常后静默失败。

### 建议
- 关键日志统一字段：时间、symbol、frequency、signal、risk_stage。

代码证据：
- `strategy/gm/callbacks.py`
- `main.py`

## 6. 非回测路径可用性（BACKTEST 之外）

### 必须
- `LIVE` 模式可启动并进入可观测运行状态。
- 无有效凭据时，`LIVE` 也应安全降级到本地仿真链路，保证非回测代码路径可执行。
- 回调链路（init/on_bar）在本地仿真与 GM 运行两条路径均可触发。

### 禁止
- 仅验证 BACKTEST 路径而忽略 LIVE/降级路径。

### 建议
- 每次发布前固定执行 BACKTEST + LIVE 启动验证。

代码证据：
- `main.py`
- `strategy/gm/callbacks.py`
- `tests/test_smoke_callbacks.py`

## 7. 企业级工程与安全编码基线

### 必须
- 采用最小权限与最小暴露原则，不在代码中硬编码密钥。
- 配置、代码、测试、报表四者保持可追溯关联。
- 对生产风险相关改动（风控阈值、执行逻辑、回调兼容）执行双人复核。

### 禁止
- 将凭据写入仓库文件。
- 无验收证据直接合并高风险改动。

### 建议
- 以 OWASP ASVS 的安全编码思想作为检查清单补充（输入处理、错误处理、日志与审计）。

### 必须
- 通过烟测：
  - `tests/test_smoke_boot.py`
  - `tests/test_smoke_callbacks.py`

  - `reports/trade_report.csv`
  - `reports/daily_report.csv`

```powershell

conda run -n minner python main.py --mode BACKTEST --config config/strategy.yaml
conda run -n minner python main.py --mode LIVE --config config/strategy.yaml
```

## 9. 可执行 PR 审核检查清单（提交前逐项勾选）

### 9.1 架构与解耦
- [ ] 未破坏 `engine/pipelines/gm/reporting` 分层边界。
- [ ] `pipelines` 未直接调用 GM SDK（下单/订阅/账户查询）。
- [ ] 流程顺序仍为 `flush -> trend -> risk -> entry -> daily`。

### 9.2 掘金免费版与回调兼容
- [ ] `run` 核心参数仍完整：`token/strategy_id/mode/backtest_start_time/backtest_end_time`（`serv_addr` 可选）。
- [ ] callback kwargs 仍按签名检测后注入。
- [ ] 凭据缺失时仍可安全降级本地仿真（含 LIVE 路径）。
- [ ] TypeError 回退仅回退 callback kwargs，不丢失核心运行参数。

### 9.3 回测与非回测可用性
- [ ] BACKTEST 可启动并产出报表。
- [ ] LIVE 可启动；无凭据时可走本地仿真链路。
- [ ] 回测窗口设置遵循“免费版最大可回测时间优先，超限回退最近完整窗口”。

### 9.4 数据与风控一致性
- [ ] 数据字段仍为 `eob/open/high/low/close/volume`，清洗逻辑未被绕过。
- [ ] 未引入未来函数风险（含 shift/rolling/merge 变更）。
- [ ] 夜盘/日盘会话阈值与配置一致（含 `instrument.sessions`）。
- [ ] 换月后挂起状态清理逻辑仍在。
- [ ] 风控优先级与停机边界未弱化。

### 9.5 证据与验收
- [ ] 已附最小证据：改动文件 + 关键日志/报表 + 验证命令结果。
- [ ] `tests/test_smoke_boot.py`、`tests/test_smoke_callbacks.py`、`tests/test_smoke_pipeline.py` 均通过。
- [ ] 文档同步完成：`README.md`、`docs/06_测试与验收.md`、本文件条款一致。
