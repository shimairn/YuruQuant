# 02 系统架构

## 模块分层

- `strategy/engine.py`：总编排，固定步骤顺序
- `strategy/pipelines/`：趋势、缠论、中枢突破、风险控制
- `strategy/gm/`：GM SDK 适配（回调、行情、订单、换月）
- `strategy/reporting/`：交易报表与日报写入
- `strategy/config_loader.py`：YAML 配置加载
- `strategy/types.py`：公共数据类型

## 关键接口

- `build_gm_callbacks(engine)`：返回 `init/on_bar/on_error/.../run_gm`
- `StrategyEngine.initialize_runtime(context)`：初始化运行态
- `StrategyEngine.process_symbol_on_5m(...)`：单品种执行主流程

## 依赖关系

- `engine` 依赖 `pipelines + gm + reporting`
- `pipelines` 不直接访问 GM SDK
- `gm` 为唯一交易与行情 IO 边界
