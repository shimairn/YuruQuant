> 用途：本文件是当前代码实现的策略规格说明，作为后续改造与验收基准。  
> 范围：仅聚焦策略本身（不含 Walk-Forward 研究流程）。  
> 状态：Implemented（已落地，2026-02-14）。

## 0. 改造目标与边界

### 0.1 目标

在不改变核心框架的前提下，把策略优化为：

1. **更简单**：减少可调参数与判定自由度。  
2. **更高效**：降低低质量交易频次，提高单位交易质量。  
3. **更稳健**：强化失败交易退出与异常行情保护。  

### 0.2 不变项（必须保持）

- 主流程顺序不变：`flush -> trend -> risk -> entry -> daily`
- 信号执行语义不变：默认“下一根执行”
- 风控优先级不变：`risk` 永远先于 `entry`
- 换月清理逻辑不变：切换主力时清理 pending 状态

## 2. 策略逻辑定稿（当前实现）

## 2.1 入场逻辑：三道门 + 两个阻断

前置条件：
- 空仓（long/short 都为 0）
- 在交易会话内

先做阻断：
1. **日内止损上限阻断**  
   - 条件：`daily_stopout_count >= max_stopouts_per_day_per_symbol`
2. **ATR 异常波动阻断**  
   - 条件：`atr_pause = atr_current / atr_ma(atr_pause_lookback)`  
   - 若 `atr_pause >= atr_pause_ratio`，当根禁止开仓  
   - `atr_pause_ratio <= 0` 视为关闭该功能

阻断触发时：
- 当前 `pending_platform` 清空
- 仅阻断开仓，不影响已有持仓风控

在空仓且允许开仓时，仅用以下 3 条规则判定候选突破：

1. **平台质量门**  
   - 条件：`width >= max(min_platform_width_atr * ATR, 8 * min_tick)`  
   - `min_platform_width_ratio` 仅兼容保留，不参与当前入场判定
2. **突破有效门**  
   - 条件：`breakout_distance >= breakout_min_distance_atr * ATR`
3. **量能门**  
   - 条件：`volume_ratio >= volume_ratio_min(day/night)`  
   - 阈值优先级：  
     1) `instrument.symbols.<csymbol>.volume_ratio_min`  
     2) `instrument.defaults.volume_ratio_min`  
     3) `strategy.breakout_volume_ratio_min`（兜底）

通过三道门后，继续走“下一根确认 + 下一根执行”机制（不改）。

开仓手数采用“固定权益比例 x 多重缩放”：
- 基础：`order_value = equity * fixed_equity_percent`
- 缩放：`size_mult = h1_size_mult * effective_risk_mult * vol_target_mult`
- 最终：`order_value = order_value * size_mult`，并受 `max_pos_size_percent` 上限约束
- 其中：
  - `h1_size_mult` 由 `h1_filter_mode` 与 1H 方向关系决定
  - `effective_risk_mult` 来自组合风控状态（normal/caution/defense/halt_recovery）
  - `vol_target_mult` 由 `target_annual_vol` 与当前 ATR 年化波动估计决定

## 2.2 风控逻辑：失败交易快速退出

总体顺序保持：组合层 -> 持仓层。

持仓层规则：
1. **hard stop**：保持现有逻辑。  
2. **time stop（新语义）**：  
   - 条件：`bars_in_trade >= time_stop_bars` 且尚未半仓（`is_half_closed == False`）  
   - 多头失败：`close <= entry_platform_zg`  
   - 空头失败：`close >= entry_platform_zd`  
   - 触发后执行平仓（`time_stop`）  
3. **1R 半仓 + 趋势跟踪退出**：  
   - 达到 1:1 后执行半仓（`split_exit_1r`），剩余仓位止损抬/降至保本附近  
   - 半仓后由 1H EMA 跟踪止损主导（`trend_following_stop`）  
4. **dynamic / trail stop**：  
   - 仅在“未半仓”阶段参与，避免和半仓后 1H 跟踪逻辑冲突

备注：
- “突破失败”依赖持仓状态中的 `entry_platform_zg/entry_platform_zd`。
- 若平台边界缺失（<=0），time stop 失败判定不触发。

## 2.3 日内止损次数上限（新增）

每个 `csymbol` 维护 `daily_stopout_count`。

- 累计场景：仅 `hard stop`、`time stop`
- 不累计场景：`portfolio forced close`
- 阻断规则：`daily_stopout_count >= max_stopouts_per_day_per_symbol` 后禁止该品种当日新开仓
- 日切重置：随交易日切换归零
